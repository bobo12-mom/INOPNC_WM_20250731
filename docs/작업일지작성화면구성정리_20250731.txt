 1. 작업일지 작성 화면 구성

  - 헤더 영역 
    (소속정보, 현장 선택, 작업 날짜 선택, 작성자 정보)
 - 현장 정보
 - 작업 내용 입력
    - 부재명 : 슬라브, 거더, 기둥, 기타 중 1택 (드롭다운) (단, 기타를 선택 시, text 입력 기능)
    - 작업공정 : 균열, 면, 마감 기타 중 1택 (드롭다운), (단, 기타를 선택 시, text 입력 기능)
    - 작업 구간 : text 입력 기능
 - 작업자 입력
    - 작업자명 : 본사가 현장에 assign한 작업자명 중 여려 명 선택 
    - 공수 : 각 assigned 작업자별로 공수 입력, 이때 공수는 0.0, 1.0, 1,5, 2.0, 2.5, 3.0 중 1택 (1.0은 8시간 근무를 의미) 
 - 사진 업로드
    - 작업전 사진 : 사진 30개까지 가능 (1장당 10MB 이하), 사진 갤러리, 사진 촬영, 파일 직접 선택 등으로 업로드 가능
    - 작업후 사진 : 사진 30개까지 가능 (1장당 10MB 이하), 사진 갤러리, 사진 촬영, 파일 직접 선택 
    - 업로드 된 사진을 삭제할 수 있는 기능 필요
- 영수증 첨부 하기
    - 일반적 영수정 첨부 정보 넣어주기. 알아서 기획해서 넣어줘요 (구분, 금액, 일자, 파일첨부 등)
- 진행 도면 업로드
    - '도면 마킹 도구'라는 기능을 별도의 화면 메뉴에서 구현할 예정입니다.
    - '도면 마킹 도구'를 활용하여 만들어진 marked 도면 (자재구간, 작업완료, 작업진행 등이 표기 됨)들 중 파일 선택해서 본 작업일지에 첨부하는 형태의 기능
- 본사에게 요청
    - 본 작업일지를 통해서 본사에게 요청하고 싶은 사항을 글로 작성할 수 있는 기능.
    - 파일 첨부하기 기능
    - 본사 화면에서는 이 정보를 취합해서 볼 수 있도록 하는 기능 필요.
-NPC-1000 자재관리
    - 자재의 입고량, 사용량, 재고량을 기록할 수 있는 기능
    - 이 정보를 기반으로 본사 담당자는 현장별로 재고 부족 여부를 판단하여 자재를 추가 생산여부 및 배송 여부를 결정할 수 있도록 하는 기능 필요.
-특이 사항 
    -텍스트로 입력하는 기능이 필요

각 섹션은 접기/펼치기 기능 필요
작업일지 작성 후, 임시 저장, 제출(저장) 버튼이 하단에 보여야 함
